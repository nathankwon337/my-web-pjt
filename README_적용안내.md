# 동기부여 시스템 적용 안내

## 무엇이 바뀌었나요

1. **gamify.js (신규 파일)** — 캐럿(크레딧)·닉네임·리그·배지·스트릭을 담당하는 공통 모듈입니다.
   서버·DB 없이 브라우저 `localStorage`에만 저장됩니다. 다른 4개 HTML 파일이 이 파일 하나를
   공유해서 불러 씁니다.

1-1. **gamify_config.js (신규 파일)** — 클라우드 동기화 URL 같은 "운영자 설정"만 담는 아주 작은 파일입니다.
   `gamify.js`가 나중에 업데이트되어도 이 파일은 그대로 두면 되므로, 설정을 다시 입력할 필요가 없습니다.

2. **문장 카드 맞추기 / 단어 짝맞추기 / 말하기 연습 / VOCA 트레이닝** (각 html 파일)
   - `<body>` 바로 다음 줄에 `<script src="gamify_config.js"></script>`와
     `<script src="gamify.js"></script>` 두 줄이 추가되었습니다.
     → 이 두 줄만으로 화면 맨 위에 "닉네임 · 리그 · 캐럿 · 스트릭"이 표시되는 얇은 바가 생깁니다.
   - 각 게임이 세트를 끝내는 지점(기존 `finishGame()` / `showDone()` 함수 안)에
     `Gamify.reportResult(...)` 호출이 한 줄~여러 줄 추가되었습니다. 게임 자체의 로직·화면·통계는
     전혀 건드리지 않았고, 결과가 나온 시점에 "보고만" 하도록 최소한으로 연결했습니다.

3. **index.html**
   - 상단에 "내 현황" 대시보드가 추가되어 닉네임·리그·캐럿·스트릭·배지 개수를 한눈에 보여줍니다.
   - 그동안 목록에 없던 **VOCA 트레이닝 카드**를 4번째로 추가했습니다(기존엔 3개 게임만 연결되어 있었어요).

## 캐럿 획득 규칙 (요약)

| 게임 | 완료 기본 캐럿 | 정확도 보너스 |
|---|---|---|
| 문장 카드 맞추기 | 10 | 90%+ → +10 / 70~89% → +5 |
| 단어 짝맞추기 | 10 | 90%+ → +10 / 70~89% → +5 |
| 말하기 연습 | 15 | 90%+ → +10 / 70~89% → +5 |
| VOCA 트레이닝(각 모듈) | 10 | 90%+ → +10 / 70~89% → +5 |

- 당일 첫 학습 +5, 연속 3일 +20 / 7일 +50 / 30일 +200
- 일일 상한 300 캐럿, 15분 내 같은 게임 재플레이 시 기본 캐럿 절반
- 리그: 🥉브론즈(0) → 🥈실버(1,000) → 🥇골드(3,000) → 💎사파이어(7,000) → 👑다이아몬드(15,000)
- 리그 승급 시 리워드 신청 코드가 자동 생성되며, 보호자가 확인 후 리워드를 지급하는 방식입니다.

## 적용(배포) 방법

기존 GitHub 저장소에 이 폴더의 파일들을 **그대로 덮어쓰기**만 하면 됩니다.

```
git add gamify.js index.html sentence_card_game.html word_match_game.html speaking_practice_game.html voca_trainer.html
git commit -m "동기부여 시스템(캐럿/닉네임/리그/배지) 적용"
git push
```

manifest.json, service-worker.js, 아이콘 파일들은 원본과 동일하므로 바꾸지 않아도 됩니다
(비교를 위해 이 폴더에도 함께 넣어두었습니다).

## 배포 전 로컬 확인 방법

1. 이 폴더를 통째로 다운로드
2. `index.html`을 더블클릭해서 브라우저로 열기 (또는 `python3 -m http.server`로 로컬 서버 실행)
3. 처음 열리면 닉네임 입력 팝업이 뜹니다 → 아무 이름이나 입력
4. 아무 게임이나 한 세트 완료 → 우측 상단에 "+N 캐럿 획득!" 알림이 뜨는지 확인
5. index.html로 돌아가서 대시보드에 캐럿·스트릭이 반영되는지 확인

## 클라우드 동기화(선택) 설정 방법 — 기기 간 이어하기 + 전체 순위

기본값은 그대로 "이 기기에만 저장"입니다. 아래를 설정하면 **다른 기기로 접속해도 기록이 이어지고, 전체 사용자 순위(리더보드)**도 볼 수 있습니다. 완전히 선택 사항이며, 설정 전/후로 게임 자체는 똑같이 동작합니다.

**중요: 이 설정은 운영자(사장님)가 딱 한 번만 하면 됩니다.** 4번 단계에서 `gamify.js` 파일 안 `DEFAULT_CLOUD_ENDPOINT`에 URL을 넣어두면, 이후 접속하는 모든 사람에게 자동으로 적용돼요 — 각자 따로 설정할 필요가 없습니다. (프로필 메뉴의 "클라우드 동기화 설정"은 이 기본값을 개인적으로 끄고 싶을 때만 쓰는 선택 기능입니다.)

### 1) 구글 스프레드시트 준비
1. [sheets.google.com](https://sheets.google.com) 에서 새 스프레드시트 생성 (이름은 자유롭게, 예: "학습게임 기록")
2. 상단 메뉴 `확장 프로그램 → Apps Script` 클릭
3. 기본으로 열려 있는 `Code.gs`의 내용을 전부 지우고, 이 폴더의 **`apps_script_Code.gs.txt`** 안의 코드를 전부 복사해서 붙여넣기
4. 저장(💾 아이콘 또는 Ctrl+S)

### 2) 웹앱으로 배포
1. 우측 상단 `배포 → 새 배포`
2. 유형 선택에서 톱니바퀴 아이콘 클릭 → **웹앱** 선택
3. 설명은 아무거나 입력, **"실행 계정"은 나(자신)**, **"액세스 권한이 있는 사용자"는 전체(Anyone)** 로 설정
4. `배포` 클릭 → 권한 승인 요청이 뜨면 본인 구글 계정으로 진행 → "고급" → "○○(안전하지 않음)으로 이동" 이 보여도 본인이 만든 스크립트이므로 그대로 진행해도 됩니다
5. 배포가 끝나면 `.../exec` 로 끝나는 **웹앱 URL**이 나옵니다 → 복사

### 3) gamify_config.js에 딱 한 번 붙여넣기 (운영자만 하면 끝)

**중요(변경됨): URL은 이제 `gamify.js`가 아니라 `gamify_config.js`라는 별도의 작은 파일에 넣습니다.**
이렇게 분리해둔 이유는, 나중에 `gamify.js`가 새 버전으로 업데이트되어도 `gamify_config.js`는
그대로 유지되기 때문에 URL을 매번 다시 넣지 않아도 되기 때문입니다.

1. `gamify_config.js` 파일을 열어서 `window.GAMIFY_CLOUD_ENDPOINT = '';` 줄을 찾기
2. 따옴표 안에 2번에서 복사한 URL을 붙여넣기 → 예: `window.GAMIFY_CLOUD_ENDPOINT = 'https://script.google.com/macros/s/AKfycb.../exec';`
3. 저장하고 GitHub 저장소에 **`gamify_config.js`와 `gamify.js` 둘 다** 반영(커밋/푸시)
   (모든 게임 HTML 파일에 이미 `<script src="gamify_config.js"></script>`가 `gamify.js`보다 먼저 불러오도록 연결되어 있습니다.)

이제 사이트에 접속하는 모든 사람에게 자동으로 클라우드 동기화가 켜집니다. 각자 "클라우드 동기화 설정"을 따로 누를 필요가 없어요. (개인이 원치 않으면 그 메뉴에서 "이 기기만 동기화 끄기"로 개별적으로 끌 수는 있습니다.)

**확인 방법:** 사이트를 열고 브라우저 개발자도구(F12) → Console 탭을 보면 `[gamify] cloud sync: ON (...)` 또는 `OFF`가 찍힙니다. `OFF`로 나온다면 `gamify_config.js`에 URL이 제대로 들어갔는지, 오탈자는 없는지 확인해주세요.

이후로는 gamify.js가 결과가 나올 때마다 이 시트에도 함께 저장하고, 접속할 때마다 시트에서 최신 기록을 가져옵니다. 시트를 열어보면 `profiles` 탭에 닉네임별로 한 줄씩 데이터가 쌓이는 걸 직접 확인하실 수 있어요(리워드 지급 확인용으로도 유용합니다).

**참고로 알아두실 점**
- 로그인/비밀번호 확인이 없는 구조라, 같은 닉네임을 다른 사람이 입력하면 기록이 섞일 수 있습니다. 가족·지인처럼 서로 아는 소규모 그룹 전제입니다.
- 두 기기에서 동시에 플레이한 직후에는 "더 높은 누적 캐럿" 쪽을 기준으로 병합됩니다(단순한 충돌 처리 방식입니다).
- 나중에 사이트 전체에서 끄고 싶으면 `DEFAULT_CLOUD_ENDPOINT`를 다시 빈 문자열 `''`로 되돌리면 됩니다.

## service-worker.js도 함께 손봤어요 (중요)

원래 서비스워커는 **"일단 예전에 저장해둔 파일부터 보여주고, 뒤에서 조용히 최신 걸로 갱신"**하는 방식이었어요.
평소엔 문제없지만, 지금처럼 `gamify.js`를 자주 업데이트하는 동안에는 **브라우저가 계속 예전 버전을 보여주는
바람에 새 기능이 안 보이거나 안 눌리는 것처럼 느껴질 수 있습니다.**

그래서 이번에 **"온라인이면 항상 최신 파일을 받아오고, 오프라인일 때만 캐시로 대체"**하는 방식으로 바꿨습니다.
이제부터는 GitHub에 새 버전을 올리면 새로고침 한 번으로 바로 반영됩니다. (오프라인 사용 기능 자체는 그대로 유지돼요.)

**지금 당장 안 눌리는 버튼이 있다면?** 아래 중 하나로 확인해보세요.
- 크롬 개발자도구(F12) → Application 탭 → Service Workers → **Unregister** → 새로고침
- 또는 시크릿 모드(Incognito)로 열어보기 (항상 최신 버전으로 열립니다)
- 그래도 안 되면 브라우저 콘솔(F12 → Console)에 빨간 에러 메시지가 있는지 확인해서 알려주세요

## 주의할 점

- 데이터는 **이 브라우저(이 기기)에만** 저장됩니다. 다른 기기·다른 브라우저로 접속하면 이어지지
  않으니, 필요하면 프로필 전환 메뉴의 "내 기록 내보내기/가져오기"로 옮겨주세요.
- 리그 강등은 없습니다. 스트릭이 끊겨도 등급은 유지됩니다(자녀 대상 서비스 특성을 고려한 설계).
- 기존 게임의 플레이 로직·UI·데이터 입력 방식(텍스트/엑셀/깃허브 불러오기)은 전혀 바뀌지 않았습니다.
