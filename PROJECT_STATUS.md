# 프로젝트 현황 요약 (다음 채팅 참고용)

마지막 업데이트: 2026-07-18

## 이 프로젝트는 무엇인가
GitHub Pages로 운영 중인 로그인 없는 외국어 학습 게임 4종(문장 카드 맞추기, 단어 짝맞추기,
말하기 연습, VOCA 트레이닝)에 동기부여(게이미피케이션) 시스템과 외부 사전 검색 기능을
추가하는 작업.

## 지금까지 적용된 것

1. **gamify.js** (신규) — 캐럿(크레딧)·닉네임·리그(브론즈~다이아몬드)·배지·스트릭을 관리하는
   공통 모듈. localStorage 기반, 서버/DB 불필요.
2. **gamify_config.js** (신규) — 클라우드 동기화 URL만 담는 설정 파일. `gamify.js`가 업데이트돼도
   이 파일은 절대 안 건드려도 됨. `window.GAMIFY_CLOUD_ENDPOINT`에 Google Apps Script 웹앱
   URL이 들어있어야 클라우드 동기화(전체 순위, 기기 간 이어하기)가 켜짐.
3. **4개 게임 HTML 전부**: `<body>` 직후 `gamify_config.js` + `gamify.js` 포함, 각 게임의
   결과 화면 시점에 `Gamify.reportResult(...)` 호출 연동됨.
4. **index.html**: 상단에 "내 현황" 대시보드(캐럿/리그/스트릭/배지) 추가, VOCA 트레이닝
   카드 포함 4개 게임 전부 연결됨.
5. **service-worker.js**: 캐시 우선 → **네트워크 우선** 방식으로 변경(v3). 이제 새 버전을
   배포하면 새로고침 한 번으로 바로 반영됨 (예전엔 캐시 때문에 업데이트가 안 보이는 문제가 있었음).
6. **클라우드 동기화(선택 기능)**: Google 스프레드시트 + Apps Script(`apps_script_Code.gs.txt`)로
   구현. 이미 설정 완료되어 시트에 데이터 저장되는 것까지 확인됨. "📊 내 순위 보기" 버튼으로
   전체 사용자 중 내 순위/백분위/리그 분포를 보여주는 통계 화면도 추가됨.
7. **word_match_game.html**: 모바일에서 카드가 가끔 두 번 눌러야 선택되던 버그 수정
   (`:hover` 규칙을 `@media (hover:hover)`로 감싸서 터치 기기에서 첫 탭에 바로 반응하도록 함).
8. **문장 카드 맞추기/말하기 연습의 `[Sentence]`, 단어 짝맞추기의 `[Word]`, VOCA의 `[Voca]`
   파일명 필터링과 발음 언어 자동 감지**는 이미 게임 자체에 구현되어 있던 기능 — 게이미피케이션
   작업과 무관하게 계속 정상 동작 중.

### 이번 세션에서 추가된 것 — 외부 사전 검색 기능

9. **`dict_lookup.js` (신규, 공용 모듈)** — 단어(또는 문장 속 단어)를 눌러 네이버/다음/
   Merriam-Webster/Google 중 선택한 사전으로 바로 연결하는 기능. `gamify.js`와 같은 방식으로
   4개 게임이 전부 `<script src="dict_lookup.js">`로 공유해서 씀 (URL 로직을 나중에 고칠 때
   이 파일 하나만 고치면 전체에 반영됨).
   - **적용 범위 (게임별)**
     - `sentence_card_game.html`: 하단 단어 은행 카드 — **길게 누르기**. 조립 중인(드래그되는)
       문장 카드는 드래그 동작과 겹쳐서 일부러 제외함.
     - `word_match_game.html`: 왼쪽(외국어) 카드만 — **길게 누르기**. 오른쪽 한글 뜻 카드는 제외.
     - `voca_trainer.html`:
       - 카드 학습(플래시카드) 표제어 — **한 번 탭**
       - 짝맞추기 모듈 왼쪽(표제어) 카드 — **길게 누르기**
       - 빈칸 채우기 · 혼동 어휘 퀴즈는 **의도적으로 제외** (사전으로 정답을 바로 찾아버릴 수 있어서)
     - `speaking_practice_game.html`: 문장 속 단어 — **한 번 탭**. 단, 띄어쓰기가 있는 언어에서만
       지원하고 중국어·일본어(글자 단위 토큰)는 제외함.
   - 각 게임 설정 화면에 "사전 찾아보기 켜기/끄기" 체크박스와 사전 선택(네이버/다음/
     Merriam-Webster/Google) 드롭다운이 추가됨. 기본값은 켜짐 + 네이버.
   - **길게 누르기 vs 한 번 탭 기준**: 이미 탭 한 번에 다른 동작(카드 선택/드래그)이 있는
     곳은 "길게 누르기"로, 탭에 아무 동작도 없는 순수 텍스트(문장, 플래시카드 표제어)는
     "한 번 탭"으로 구현함 — 후자에 롱프레스를 썼더니 OS의 텍스트 선택(복사) 팝업과
     충돌하는 문제가 있었음 (10번 항목 참고).

10. **네이버 사전 URL 문제 해결** — 예전에 쓰던 `dict.naver.com/search.nhn?dicQuery=` 방식이
    리다이렉트되며 검색어가 유실되는 문제가 있어 아래처럼 교체함:
    - 영어: `https://en.dict.naver.com/#/search?query=` — **실기기로 확인됨**
    - 독일어·베트남어: `https://dict.naver.com/{언어코드}kodict/#/search?query=` —
      **실기기로 확인됨** (예: 독일어 `dekodict`, 베트남어 `vikodict`)
    - 스페인어·프랑스어·이탈리아어·러시아어·포르투갈어·태국어·인도네시아어: 위와 같은
      `{코드}kodict` 규칙을 적용해뒀지만 **개별적으로 실기기 확인은 아직 안 됨**
      (독일어·베트남어 두 언어가 같은 규칙이라는 것만 확인된 상태에서 유추 적용).
    - 그 외 언어(일본어·중국어 등)는 확인된 URL 규칙이 없어 안전하게 Google 정의 검색으로 대체됨.
    - **다음사전은 독일어·스페인어 사전 자체가 없음** (번역기만 지원, 사전은 미지원) — 이
      두 언어를 다음사전으로 선택해도 자동으로 네이버(또는 Google)로 대체되고, 팝업에
      "다음사전은 이 언어를 지원하지 않아 ○○로 연결했어요" 안내가 뜸.

11. **"사전 찾아본 다음 카드가 다시 안 눌리는(두 번 눌러야 선택되는)" 버그 수정** — 원인 추정:
    모바일 브라우저가 롱프레스 직후의 합성 `click` 이벤트를 항상 안정적으로 만들어주지는
    않아서, 롱프레스 직후 첫 선택 탭이 씹히던 것으로 보임. `dict_lookup.js`를 `click` 이벤트에
    의존하지 않는 구조로 재설계 — `pointerdown/pointermove/pointerup`만으로 "탭인지, 롱프레스인지,
    드래그인지"를 직접 판별하고, 카드 선택 로직도 `onTap` 콜백에서 바로 실행하도록 옮김.
    또한 `contextmenu` 이벤트 차단 + `user-select:none`/`-webkit-touch-callout:none` CSS를
    자동으로 붙여서, OS 자체의 길게 누르기(텍스트 선택/복사 팝업) 제스처와의 충돌도 줄임.

12. **gamify.js 상단바 모바일 표시 개선** — 화면이 좁을 때(480px 이하) 닉네임 텍스트가
    공간이 부족해 글자가 세로로 깨져 보이던 문제 수정. 미디어쿼리를 추가해 좁은 화면에서는
    이니셜 아바타만 보이게 하고, PC/태블릿처럼 공간이 충분하면 기존처럼 닉네임이 그대로
    보이도록 함. 아바타에 `title` 속성을 추가해 눌러 유지하거나 마우스를 올리면 이름을 확인할 수 있음.

## 되돌린 것 (실패한 시도)

- **speaking_practice_game.html에 마이크 볼륨 반응 효과(음성 크기에 따라 링이 커지는 효과)를
  추가했다가 되돌림.** 원인 추정: `getUserMedia`로 별도 마이크 스트림을 열면서 일부 모바일
  브라우저에서 SpeechRecognition(음성인식)과 충돌해서 음성인식 자체가 안 되는 문제 발생.
  **다시 시도한다면**: SpeechRecognition과 동시에 별도 마이크 스트림을 열지 않는 방식(예:
  interim 인식 결과의 길이/타이밍만으로 시각효과를 유사하게 흉내내는 방식)을 고려할 것.

## 확인이 더 필요한 부분 (다음에 이어서 볼 것)

- **네이버 사전 `{코드}kodict` 규칙의 나머지 언어 검증**: 스페인어·프랑스어·이탈리아어·
  러시아어·포르투갈어·태국어·인도네시아어 문장으로 실제 사전 연결이 되는지 테스트 필요.
  틀린 언어가 있으면 어떤 언어의 어떤 주소가 맞는지(브라우저 주소창 캡처) 알려주면 그
  언어만 콕 집어 고치면 됨.
- 사전 검색 기능을 다른 화면(예: 결과/복습 화면)까지 넓힐지는 아직 정해지지 않음.

## 파일 간 의존 관계 (중요)

- 게임 4개 HTML은 전부 같은 `gamify.js` / `gamify_config.js` **그리고 `dict_lookup.js`**를
  공유함 → 이 셋 중 하나만 바꿔도 4개 게임에 다 영향감. GitHub Pages에 배포할 때
  `dict_lookup.js`도 같은 폴더에 반드시 함께 올려야 함 (빠지면 사전 기능이 있는 화면에서
  `DictLookup is not defined` 오류가 남).
- `gamify_config.js`는 **사이트 소유자만 한 번 채우는 파일**이라 업데이트 시 덮어쓰지 않도록 주의.
- 새 `gamify.js`를 받을 때마다 `gamify_config.js`는 그대로 두면 됨 (URL 재입력 불필요하도록
  일부러 분리해둔 구조).
